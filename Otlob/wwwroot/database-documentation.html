🗄️ What is a Database?

Definition

A database is an organized collection of structured data stored electronically in a computer system. It allows for efficient storage, retrieval, modification, and management of data through a Database Management System (DBMS).

🔑 Key Concepts

Data Persistence: Data remains available even after the application or server restarts.

Structured Storage: Data is organized in tables, rows, and columns for easy access.

ACID Properties: Ensures reliability through Atomicity, Consistency, Isolation, and Durability.

Query Language: SQL (Structured Query Language) allows complex data operations.

Types of Databases

Type Description Examples Use Cases
Relational (RDBMS) Tables with relationships, SQL-based SQL Server, MySQL, PostgreSQL Business applications, E-commerce
NoSQL Document, key-value, or graph-based MongoDB, Redis, Cassandra Big data, Real-time analytics
NewSQL Combines SQL with NoSQL scalability CockroachDB, Google Spanner Distributed systems
In-Memory Data stored in RAM for speed Redis, Memcached Caching, Session management

Why Use a Database?

💾 Data Persistence

Store data permanently and reliably, surviving application restarts and system reboots.

🔍 Efficient Queries

Quickly search, filter, and retrieve data using optimized query languages.

🔐 Data Integrity

Enforce rules, constraints, and relationships to maintain data accuracy.

👥 Concurrent Access

Multiple users can access and modify data simultaneously without conflicts.

🔄 Transaction Support

Group operations together ensuring all-or-nothing execution (ACID compliance).

📊 Data Analysis

Perform complex aggregations, reporting, and business intelligence operations.

🛢️ Microsoft SQL Server

What is Microsoft SQL Server?

Microsoft SQL Server is a relational database management system (RDBMS) developed by Microsoft. It's one of the most popular and powerful database systems in the world, designed to handle workloads ranging from small single-machine applications to large internet-facing applications with many concurrent users.

🔗 ASP.NET Core Integration with SQL Server

How They Work Together

ASP.NET Core and SQL Server form a powerful combination for building modern web applications. The integration is facilitated primarily through Entity Framework Core (EF Core), an Object-Relational Mapper (ORM) that allows developers to work with databases using .NET objects.

Architecture Stack

ASP.NET Core Application (MVC / Web API)
⬇️
Entity Framework Core (ORM Layer)
⬇️
SQL Server Database Provider
⬇️
Microsoft SQL Server Database

Entity Framework Core (EF Core)

EF Core is a modern object-database mapper that enables .NET developers to work with databases using .NET objects, eliminating the need for most data-access code. It supports LINQ queries, change tracking, updates, and schema migrations.

Key Integration Features

🎯 Object-Relational Mapping

Map C# classes to database tables automatically, eliminating boilerplate SQL code.

📝 LINQ Queries

Write type-safe queries in C# that are translated to SQL at runtime.

🔄 Change Tracking

Automatically detect changes to entities and generate appropriate SQL commands.

🚀 Migrations

Code-first approach to evolve database schema alongside application code.

⚡ Performance

Compiled queries, connection pooling, and efficient SQL generation.

🧪 Testability

Easy mocking and in-memory database providers for unit testing.

Connection Configuration

Setting up SQL Server with ASP.NET Core involves configuring the connection string and registering the DbContext:

// appsettings.json { "ConnectionStrings": { "DefaultConnection": "Server=.;Database=OtlobDB;Trusted_Connection=True;TrustServerCertificate=True" } } // Program.cs var connectionString = builder.Configuration.GetConnectionString("DefaultConnection"); builder.Services.AddDbContext<ApplicationDbContext>(options => options.UseSqlServer(connectionString));

DbContext Example

public class ApplicationDbContext : DbContext { public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options) : base(options) { } // DbSet properties represent tables public DbSet<Restaurant> Restaurants { get; set; } public DbSet<Order> Orders { get; set; } public DbSet<Meal> Meals { get; set; } protected override void OnModelCreating(ModelBuilder modelBuilder) { // Configure entity relationships and constraints modelBuilder.ApplyConfigurationsFromAssembly(typeof(ApplicationDbContext).Assembly); } }

Repository Pattern Implementation

// Repository interface public interface IRepository<T> where T : class { Task<IEnumerable<T>> GetAllAsync(); Task<T?> GetByIdAsync(object id); Task AddAsync(T entity); void Update(T entity); void Delete(T entity); } // Repository implementation public class Repository<T> : IRepository<T> where T : class { private readonly ApplicationDbContext _context; private readonly DbSet<T> _dbSet; public Repository(ApplicationDbContext context) { _context = context; _dbSet = context.Set<T>(); } public async Task<IEnumerable<T>> GetAllAsync() => await _dbSet.ToListAsync(); public async Task<T?> GetByIdAsync(object id) => await _dbSet.FindAsync(id); public async Task AddAsync(T entity) => await _dbSet.AddAsync(entity); public void Update(T entity) => _dbSet.Update(entity); public void Delete(T entity) => _dbSet.Remove(entity); }

💻 Implementation Guide

Step 1: Install SQL Server

Options:

  • SQL Server Express: Download from Microsoft's website (free)
  • SQL Server Developer: Full features for development (free)
  • Azure SQL Database: Cloud-based, no installation needed
  • LocalDB: Lightweight version for development (included with Visual Studio)

Step 2: Install NuGet Packages

# Install Entity Framework Core SQL Server provider dotnet add package Microsoft.EntityFrameworkCore.SqlServer # Install EF Core tools for migrations dotnet add package Microsoft.EntityFrameworkCore.Tools # Install design-time support dotnet add package Microsoft.EntityFrameworkCore.Design

Step 3: Create Entity Classes

public class Restaurant { public int Id { get; set; } public string Name { get; set; } = string.Empty; public string Description { get; set; } = string.Empty; public bool IsActive { get; set; } // Navigation properties public ICollection<Meal> Meals { get; set; } = new List<Meal>(); public ICollection<Order> Orders { get; set; } = new List<Order>(); }

Step 4: Configure DbContext

public class ApplicationDbContext : DbContext { public ApplicationDbContext(DbContextOptions<ApplicationDbContext> options) : base(options) { } public DbSet<Restaurant> Restaurants { get; set; } public DbSet<Meal> Meals { get; set; } public DbSet<Order> Orders { get; set; } protected override void OnModelCreating(ModelBuilder modelBuilder) { // Configure entity relationships modelBuilder.Entity<Restaurant>() .HasMany(r => r.Meals) .WithOne(m => m.Restaurant) .HasForeignKey(m => m.RestaurantId) .OnDelete(DeleteBehavior.Cascade); // Apply configurations from separate files modelBuilder.ApplyConfigurationsFromAssembly(typeof(ApplicationDbContext).Assembly); } }

Step 5: Configure Connection String

// appsettings.json { "ConnectionStrings": { "DefaultConnection": "Server=.;Database=OtlobDB;Trusted_Connection=True;TrustServerCertificate=True", "AzureConnection": "Server=tcp:myserver.database.windows.net;Database=OtlobDB;User ID=admin;Password=***" } } // Program.cs var connectionString = builder.Configuration.GetConnectionString("DefaultConnection") ?? throw new InvalidOperationException("Connection string not found"); builder.Services.AddDbContext<ApplicationDbContext>(options => { options.UseSqlServer(connectionString); options.EnableSensitiveDataLogging(builder.Environment.IsDevelopment()); });

Step 6: Create and Apply Migrations

# Create initial migration dotnet ef migrations add InitialCreate # Apply migration to database dotnet ef database update # Add new migration after model changes dotnet ef migrations add AddNewFeature # Rollback to specific migration dotnet ef database update PreviousMigrationName # Remove last migration (if not applied) dotnet ef migrations remove

Step 7: Implement Repository Pattern

// Service registration builder.Services.AddScoped<IUnitOfWork, UnitOfWork>(); builder.Services.AddScoped(typeof(IRepository<>), typeof(Repository<>)); // Usage in controller/service public class RestaurantService { private readonly IUnitOfWork _unitOfWork; public RestaurantService(IUnitOfWork unitOfWork) { _unitOfWork = unitOfWork; } public async Task<Restaurant?> GetRestaurantAsync(int id) { return await _unitOfWork.Restaurants .Include(r => r.Meals) .FirstOrDefaultAsync(r => r.Id == id); } public async Task AddRestaurantAsync(Restaurant restaurant) { await _unitOfWork.Restaurants.AddAsync(restaurant); await _unitOfWork.CompleteAsync(); } }

Step 8: Query Optimization

// Eager loading - Load related data in single query var restaurants = await _context.Restaurants .Include(r => r.Meals) .Include(r => r.Orders) .ThenInclude(o => o.OrderDetails) .ToListAsync(); // Projection - Select only needed columns var restaurantNames = await _context.Restaurants .Select(r => new { r.Id, r.Name }) .ToListAsync(); // AsNoTracking - Read-only queries (faster) var readOnlyData = await _context.Restaurants .AsNoTracking() .ToListAsync(); // Compiled queries - Pre-compile for reuse private static readonly Func<ApplicationDbContext, int, Task<Restaurant?>> GetRestaurantById = EF.CompileAsyncQuery( (ApplicationDbContext context, int id) => context.Restaurants.FirstOrDefault(r => r.Id == id));

Best Practices

📋 Use Migrations

Version control your database schema changes for reliable deployments.

🔍 Index Properly

Add indexes on foreign keys and frequently queried columns.

⚡ Optimize Queries

Use AsNoTracking, projections, and avoid N+1 query problems.

🔒 Secure Connections

Use encrypted connections, parameterized queries, and proper authentication.

💾 Handle Transactions

Use Unit of Work pattern for managing complex operations.

🧪 Test Thoroughly

Use in-memory providers for unit tests, real database for integration tests.
